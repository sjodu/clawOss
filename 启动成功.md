## 工作流程

### 自动运行（每 5 分钟）
1. **Step 0**: 健康检查、预算检查、always-on subagent 管理
2. **Step 1**: 恢复卡住的任务
3. **Step 2**: 选择新工作（优先新 PR）
4. **Step 3**: 发现 GitHub issues (scout subagent)
5. **Step 4**: 分类和评分
6. **Step 5**: 生成实现子任务
7. **Step 6**: 处理子任务结果
8. **Step 7**: 清理和报告

### Always-On Subagents (4个)
- **Scout**: 持续发现 issues
- **PR Monitor Scan**: 快速扫描 PR 状态
- **PR Monitor Deep**: 深度分析评论
- **PR Analyst**: 策略分析和 repo 评估

### Implementation Slots (10个)
- 用于新 PR 实现和 follow-up 响应
- 优先填充新 PR，然后处理 follow-up
