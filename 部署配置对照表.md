# ClawOSS 部署配置对照表

## 核心配置参数

| 配置项 | 说明 | GitHub Actions | 云服务器（推荐） | 云服务器（最低配） |
|--------|------|----------------|------------------|-------------------|
| **运行频率** | 任务触发间隔 | 每 4 小时 | 每 15 分钟 | 每 30 分钟 |
| **单次超时** | 单次运行最长时间 | 50 分钟 | 无限制 | 无限制 |
| **心跳间隔** | HEARTBEAT 循环周期 | 15 分钟 | 5 分钟 | 10 分钟 |
| **并发任务数** | 同时处理的 PR 数量 | 2 个 | 5 个 | 2 个 |
| **预算上限** | 每月最大花费（USD） | $3.0 | $10.0 | $5.0 |

## 环境变量配置

### 必需配置（所有部署方式）

```bash
# LLM 配置
LLM_PROVIDER=deepseek              # 模型提供商
LLM_MODEL=deepseek-chat            # 模型名称
LLM_API_KEY=sk-xxx                 # API 密钥
LLM_BASE_URL=https://api.deepseek.com  # API 端点
LLM_COST_INPUT=0.14                # 输入成本（$/M tokens）
LLM_COST_OUTPUT=0.28               # 输出成本（$/M tokens）

# GitHub 配置
GITHUB_TOKEN=ghp_xxx               # GitHub Personal Access Token
GITHUB_USERNAME=your-username      # GitHub 用户名
GITHUB_EMAIL=your@email.com        # GitHub 邮箱

# 预算控制
BUDGET_MAX_USD=5.0                 # 预算上限（根据部署方式调整）
```

### 可选配置（运行时优化）

```bash
# 心跳配置
HEARTBEAT_INTERVAL_MINUTES=15      # 心跳间隔（分钟）

# 并发控制
MAX_CONCURRENT_TASKS=2             # 最大并发任务数

# 日志级别
LOG_LEVEL=info                     # debug | info | warn | error
```

## 部署方式对比

### 1. GitHub Actions（免费额度有限）

**适用场景：**
- 测试和验证
- 低频率运行（每天 6 次）
- 预算有限

**配置要点：**
```yaml
# .github/workflows/clawoss-runner.yml
schedule:
  - cron: '0 */4 * * *'  # 每 4 小时

timeout-minutes: 50
env:
  HEARTBEAT_INTERVAL_MINUTES: 15
  MAX_CONCURRENT_TASKS: 2
  BUDGET_MAX_USD: 3.0
```

**成本估算：**
- 每次运行：8-10 分钟
- 每天运行：6 次
- 每月消耗：~1800 分钟（在 2000 分钟免费额度内）
- LLM 成本：~$3/月

---

### 2. 云服务器 - 推荐配置（1核2GB）

**适用场景：**
- 生产环境
- 高频率运行（每 15 分钟）
- 需要稳定性

**服务器规格：**
- CPU：1 核
- 内存：2GB
- 磁盘：20GB
- 带宽：1Mbps

**配置要点：**
```bash
# 环境变量（.env）
HEARTBEAT_INTERVAL_MINUTES=5
MAX_CONCURRENT_TASKS=5
BUDGET_MAX_USD=10.0

# 系统服务（systemd）
Restart=always
RestartSec=30
```

**成本估算：**
- 服务器：¥50-80/月（阿里云/腾讯云）
- LLM 成本：~$10/月
- 总计：~¥150/月

---

### 3. 云服务器 - 最低配置（1核1GB）

**适用场景：**
- 预算极度有限
- 中频率运行（每 30 分钟）
- 可接受偶尔重启

**服务器规格：**
- CPU：1 核
- 内存：1GB + 2GB Swap
- 磁盘：20GB
- 带宽：1Mbps

**配置要点：**
```bash
# 环境变量（.env）
HEARTBEAT_INTERVAL_MINUTES=10
MAX_CONCURRENT_TASKS=2
BUDGET_MAX_USD=5.0

# 添加 Swap（必需）
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

**成本估算：**
- 服务器：¥30-50/月
- LLM 成本：~$5/月
- 总计：~¥80/月

## 快速切换指南

### 从 GitHub Actions 切换到云服务器

1. **复制配置文件**
   ```bash
   scp .env root@your-server:/root/clawoss/
   scp -r workspace/memory root@your-server:/root/clawoss/workspace/
   ```

2. **调整环境变量**
   ```bash
   # 在服务器上编辑 .env
   vim /root/clawoss/.env
   
   # 修改以下参数
   HEARTBEAT_INTERVAL_MINUTES=5    # 从 15 改为 5
   MAX_CONCURRENT_TASKS=5          # 从 2 改为 5
   BUDGET_MAX_USD=10.0             # 从 3.0 改为 10.0
   ```

3. **启动服务**
   ```bash
   cd /root/clawoss
   bash scripts/restart.sh
   ```

### 从云服务器切换到 GitHub Actions

1. **备份数据**
   ```bash
   scp root@your-server:/root/clawoss/workspace/memory/* ./workspace/memory/
   ```

2. **配置 GitHub Secrets**
   - 进入仓库 Settings → Secrets and variables → Actions
   - 添加所有环境变量（见上方"必需配置"）

3. **推送代码**
   ```bash
   git add .github/workflows/clawoss-runner.yml
   git commit -m "Enable GitHub Actions deployment"
   git push origin main
   ```

## 配置模板文件

### GitHub Actions 配置模板

保存为 `.github/workflows/clawoss-runner.yml`：

```yaml
name: ClawOSS Autonomous Runner

on:
  schedule:
    - cron: '0 */4 * * *'  # 每 4 小时
  workflow_dispatch:

jobs:
  run-clawoss:
    runs-on: ubuntu-latest
    timeout-minutes: 50
    
    env:
      # 从 Secrets 读取
      LLM_PROVIDER: ${{ secrets.LLM_PROVIDER }}
      LLM_MODEL: ${{ secrets.LLM_MODEL }}
      LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
      LLM_BASE_URL: ${{ secrets.LLM_BASE_URL }}
      LLM_COST_INPUT: ${{ secrets.LLM_COST_INPUT }}
      LLM_COST_OUTPUT: ${{ secrets.LLM_COST_OUTPUT }}
      GITHUB_TOKEN: ${{ secrets.GH_TOKEN }}
      GITHUB_USERNAME: ${{ secrets.GH_USERNAME }}
      GITHUB_EMAIL: ${{ secrets.GH_EMAIL }}
      
      # 运行时配置
      HEARTBEAT_INTERVAL_MINUTES: 15
      MAX_CONCURRENT_TASKS: 2
      BUDGET_MAX_USD: 3.0
    
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Install OpenClaw
        run: npm install -g openclaw
      
      - name: Run ClawOSS
        run: bash scripts/restart.sh
```

## 常见问题

**Q: GitHub Actions 免费额度用完了怎么办？**
A: 切换到云服务器部署，或降低运行频率（每 6-8 小时）。

**Q: 云服务器内存不足怎么办？**
A: 添加 Swap 空间（见"最低配置"部分），或升级到 2GB 内存。

**Q: 如何同时使用两种部署方式？**
A: 不推荐。建议选择一种主要方式，另一种作为备份。

**Q: 预算控制是否跨部署方式共享？**
A: 不共享。每种部署方式独立追踪预算（通过 `workspace/memory/budget-spent.txt`）。
