# ClawOSS 线上部署指南

## 部署方案概览

ClawOSS 支持多种线上部署方式，根据你的需求选择：

| 方案 | 适用场景 | 成本 | 难度 |
|------|---------|------|------|
| 1. VPS/云服务器 | 长期运行，完全控制 | $5-20/月 | ⭐⭐ |
| 2. Docker 容器 | 快速部署，易于迁移 | $5-20/月 | ⭐⭐ |
| 3. Kubernetes | 大规模，高可用 | $50+/月 | ⭐⭐⭐⭐ |
| 4. 无服务器 | 按需运行，低成本 | $1-5/月 | ⭐⭐⭐ |

## 方案 1：VPS/云服务器部署（推荐）

### 1.1 选择云服务商

**推荐服务商：**
- **阿里云 ECS**（国内访问快）
- **腾讯云 CVM**（国内访问快）
- **AWS EC2**（全球覆盖）
- **DigitalOcean Droplet**（简单易用）
- **Vultr**（性价比高）

**最低配置要求：**
```
CPU: 2 核
内存: 4GB
硬盘: 40GB SSD
带宽: 1Mbps
系统: Ubuntu 22.04 LTS
```

**推荐配置：**
```
CPU: 4 核
内存: 8GB
硬盘: 80GB SSD
带宽: 5Mbps
系统: Ubuntu 22.04 LTS
月费: ~$20
```

### 1.2 部署步骤

#### Step 1: 连接到服务器

```bash
# 使用 SSH 连接
ssh root@your-server-ip

# 或使用密钥
ssh -i ~/.ssh/your-key.pem ubuntu@your-server-ip
```

#### Step 2: 安装依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 24
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 Git
sudo apt install -y git

# 安装 GitHub CLI
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install -y gh

# 安装 Python 3
sudo apt install -y python3 python3-pip

# 安装 OpenClaw
npm install -g openclaw
```

#### Step 3: 克隆项目

```bash
# 创建工作目录
mkdir -p ~/clawoss
cd ~/clawoss

# 克隆项目（或上传你的项目文件）
git clone https://github.com/your-username/ClawOSS-6-release.git
cd ClawOSS-6-release

# 或使用 scp 上传
# 本地执行：scp -r /Users/jiangjiang/Downloads/ClawOSS-6-release root@your-server-ip:~/clawoss/
```

#### Step 4: 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置
nano .env
```

**配置内容：**
```bash
# LLM Configuration
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
LLM_API_KEY=sk-your-deepseek-key
LLM_BASE_URL=https://api.deepseek.com
LLM_COST_INPUT=0.14
LLM_COST_OUTPUT=0.28

# Budget Control
BUDGET_MAX_USD=50.0

# GitHub Configuration
GITHUB_TOKEN=ghp_your-github-token
GITHUB_USERNAME=your-username
GITHUB_EMAIL=your-email@example.com

# Dashboard (Optional)
DASHBOARD_URL=https://clawoss-dashboard.vercel.app
```

#### Step 5: 配置 GitHub 认证

```bash
# 使用 GitHub token 认证
echo $GITHUB_TOKEN | gh auth login --with-token

# 配置 Git
git config --global user.name "your-username"
git config --global user.email "your-email@example.com"
```

#### Step 6: 启动系统

```bash
# 赋予执行权限
chmod +x scripts/*.sh

# 启动系统
bash scripts/restart.sh
```

#### Step 7: 配置为系统服务（保持后台运行）

创建 systemd 服务文件：

```bash
sudo nano /etc/systemd/system/clawoss.service
```

**服务配置：**
```ini
[Unit]
Description=ClawOSS Autonomous Agent
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/clawoss/ClawOSS-6-release
Environment="PATH=/usr/local/bin:/usr/bin:/bin"
ExecStart=/usr/bin/bash /root/clawoss/ClawOSS-6-release/scripts/restart.sh
Restart=always
RestartSec=10
StandardOutput=append:/var/log/clawoss.log
StandardError=append:/var/log/clawoss-error.log

[Install]
WantedBy=multi-user.target
```

**启动服务：**
```bash
# 重载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start clawoss

# 设置开机自启
sudo systemctl enable clawoss

# 查看状态
sudo systemctl status clawoss

# 查看日志
sudo journalctl -u clawoss -f
```

## 方案 4：Kubernetes 部署（企业级）

### 4.1 创建 Kubernetes 配置

创建 `k8s/deployment.yaml`：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: clawoss
spec:
  replicas: 1
  selector:
    matchLabels:
      app: clawoss
  template:
    metadata:
      labels:
        app: clawoss
    spec:
      containers:
      - name: clawoss
        image: your-registry/clawoss:latest
        env:
        - name: LLM_PROVIDER
          valueFrom:
            secretKeyRef:
              name: clawoss-secrets
              key: llm-provider
        - name: LLM_API_KEY
          valueFrom:
            secretKeyRef:
              name: clawoss-secrets
              key: llm-api-key
        - name: GITHUB_TOKEN
          valueFrom:
            secretKeyRef:
              name: clawoss-secrets
              key: github-token
        resources:
          requests:
            memory: "4Gi"
            cpu: "2"
          limits:
            memory: "8Gi"
            cpu: "4"
---
apiVersion: v1
kind: Secret
metadata:
  name: clawoss-secrets
type: Opaque
stringData:
  llm-provider: deepseek
  llm-api-key: sk-your-key
  github-token: ghp-your-token
```

### 4.2 部署到 Kubernetes

```bash
# 应用配置
kubectl apply -f k8s/deployment.yaml

# 查看状态
kubectl get pods
kubectl logs -f clawoss-xxx

# 扩容
kubectl scale deployment clawoss --replicas=3
```

## 安全配置

### 1. 防火墙设置

```bash
# 安装 UFW
sudo apt install -y ufw

# 允许 SSH
sudo ufw allow 22/tcp

# 允许 OpenClaw Gateway（如果需要外部访问）
sudo ufw allow 18789/tcp

# 启用防火墙
sudo ufw enable
```

### 2. 配置 HTTPS（使用 Nginx + Let's Encrypt）

```bash
# 安装 Nginx
sudo apt install -y nginx certbot python3-certbot-nginx

# 配置 Nginx
sudo nano /etc/nginx/sites-available/clawoss
```

**Nginx 配置：**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/clawoss /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

# 获取 SSL 证书
sudo certbot --nginx -d your-domain.com
```

### 3. 环境变量加密

```bash
# 使用 age 加密敏感文件
sudo apt install -y age

# 生成密钥
age-keygen -o ~/.age-key.txt

# 加密 .env
age -r $(cat ~/.age-key.txt | grep public | cut -d: -f2) .env > .env.age

# 解密
age -d -i ~/.age-key.txt .env.age > .env
```

## 监控和维护

### 1. 日志管理

```bash
# 查看系统日志
sudo journalctl -u clawoss -f

# 查看 OpenClaw 日志
openclaw logs --follow

# 查看预算状态
bash scripts/budget-status.sh

# 查看 PR 状态
gh search prs --author BillionClaw --state open
```

### 2. 自动备份

创建备份脚本 `scripts/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/backup/clawoss"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份配置和数据
tar -czf $BACKUP_DIR/clawoss_$DATE.tar.gz \
    .env \
    workspace/memory/ \
    ~/.openclaw/

# 保留最近 7 天的备份
find $BACKUP_DIR -name "clawoss_*.tar.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/clawoss_$DATE.tar.gz"
```

**设置定时备份：**
```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨 2 点备份
0 2 * * * /root/clawoss/ClawOSS-6-release/scripts/backup.sh
```

### 3. 监控脚本

创建 `scripts/monitor.sh`：

```bash
#!/bin/bash

# 检查 Gateway 状态
if ! openclaw gateway status | grep -q "running"; then
    echo "Gateway is down! Restarting..."
    openclaw gateway restart
fi

# 检查预算
SPENT=$(cat workspace/memory/budget-spent.txt)
MAX=$(cat workspace/memory/budget-max.txt)
if (( $(echo "$SPENT >= $MAX" | bc -l) )); then
    echo "Budget exhausted! Pausing system..."
    openclaw gateway stop
fi

# 检查磁盘空间
DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}' | sed 's/%//')
if [ $DISK_USAGE -gt 80 ]; then
    echo "Disk usage high: ${DISK_USAGE}%"
fi
```

**设置定时监控：**
```bash
# 每 5 分钟检查一次
*/5 * * * * /root/clawoss/ClawOSS-6-release/scripts/monitor.sh
```

## 故障排查

### 常见问题

1. **Gateway 无法启动**
   ```bash
   # 检查端口占用
   lsof -i :18789
   
   # 重启 Gateway
   openclaw gateway stop
   openclaw gateway start
   ```

2. **预算耗尽**
   ```bash
   # 重置预算
   bash scripts/budget-reset.sh
   ```

3. **GitHub 认证失败**
   ```bash
   # 重新认证
   gh auth login
   ```

4. **磁盘空间不足**
   ```bash
   # 清理日志
   openclaw logs --clear
   
   # 清理 Docker
   docker system prune -a
   ```
