# ClawOSS 模型配置灵活化改造分析

## 一、需求分析

### 1.1 核心需求
1. **模型配置灵活化**: 通过环境变量配置任意主流大模型，无需修改代码
2. **预算控制**: 设置 token 总预算，达到预算时自动暂停服务
3. **Dashboard 正常运行**: 实时反映系统运行情况
4. **避免硬编码**: 不通过写死流程实现目标

### 1.2 验收标准
- 可标准化部署并自主运行
- 可任意配置主流大模型
- 正常提交 PR 到高质量项目
- 跟进反馈直到 PR 合并
- 预算控制生效

### 2.2 问题总结
1. **15+ 处硬编码**: 分散在配置、脚本、钩子中
2. **无模型抽象**: 没有统一的模型配置接口
3. **成本计算固定**: 仅支持 Kimi 的定价
4. **无预算控制**: 没有 token 预算限制机制
5. **启动依赖特定模型**: 必须配置 MiniMax 或 Kimi 才能启动

## 三、改造方案设计

### 3.1 架构设计原则
1. **配置驱动**: 所有模型配置通过环境变量和配置文件
2. **提供商抽象**: 支持 OpenAI、Anthropic、OpenRouter、Azure 等主流提供商
3. **成本透明**: 自动从配置读取成本，支持自定义
4. **预算控制**: 全局 token 预算监控和熔断
5. **向后兼容**: 保持现有 MiniMax/Kimi 配置可用

### 3.3 预算控制系统

#### 3.3.1 预算追踪器设计
```typescript
// workspace/hooks/budget-tracker/handler.ts
interface BudgetState {
  maxBudgetUsd: number;
  currentSpendUsd: number;
  warningThreshold: number;
  isExceeded: boolean;
  lastCheckTime: number;
}

class BudgetTracker {
  private state: BudgetState;
  private stateFile = '~/.openclaw/budget-state.json';
  
  // 累积成本
  accumulate(inputTokens: number, outputTokens: number, costConfig: CostConfig): void {
    const cost = (inputTokens * costConfig.input + outputTokens * costConfig.output) / 1_000_000;
    this.state.currentSpendUsd += cost;
    this.save();
    
    // 检查预算
    if (this.state.currentSpendUsd >= this.state.maxBudgetUsd) {
      this.triggerBudgetExceeded();
    } else if (this.state.currentSpendUsd >= this.state.maxBudgetUsd * this.state.warningThreshold) {
      this.triggerBudgetWarning();
    }
  }
  
  // 预算超限处理
  private triggerBudgetExceeded(): void {
    this.state.isExceeded = true;
    this.save();
    
    // 停止网关
    exec('openclaw gateway stop');
    
    // 发送通知
    this.notifyDashboard({
      type: 'budget_exceeded',
      currentSpend: this.state.currentSpendUsd,
      maxBudget: this.state.maxBudgetUsd
    });
    
    console.error(`[BUDGET] EXCEEDED: $${this.state.currentSpendUsd.toFixed(2)} / $${this.state.maxBudgetUsd}`);
  }
  
  // 重置预算（手动操作）
  reset(): void {
    this.state.currentSpendUsd = 0;
    this.state.isExceeded = false;
    this.save();
  }
}
```

#### 3.3.2 预算检查集成点
1. **dashboard-reporter hook**: 每次 token 使用后累积
2. **heartbeat 开始**: 检查预算状态，超限则拒绝执行
3. **子代理生成前**: 检查预算，超限则跳过生成
4. **定期检查**: 每 5 分钟检查一次预算状态

### 3.4 Dashboard 成本计算改造

#### 3.4.1 动态成本配置
```typescript
// workspace/hooks/dashboard-reporter/handler.ts
interface CostConfig {
  input: number;    // 每百万 tokens 成本
  output: number;
  cacheRead?: number;
  cacheWrite?: number;
}

function loadCostConfig(): CostConfig {
  return {
    input: parseFloat(process.env.LLM_COST_INPUT || '10.0'),
    output: parseFloat(process.env.LLM_COST_OUTPUT || '30.0'),
    cacheRead: parseFloat(process.env.LLM_COST_CACHE_READ || '0'),
    cacheWrite: parseFloat(process.env.LLM_COST_CACHE_WRITE || '0')
  };
}

function calculateCost(inputTokens: number, outputTokens: number): number {
  const config = loadCostConfig();
  return (inputTokens * config.input + outputTokens * config.output) / 1_000_000;
}
```

## 六、风险与挑战

### 6.1 技术风险
1. **OpenClaw 兼容性**: 需要验证 OpenClaw 是否支持所有提供商
2. **API 差异**: 不同提供商的 API 格式可能有差异
3. **成本计算准确性**: 缓存、批处理等特殊计费需要特殊处理
4. **预算竞态条件**: 多个子代理并发时的预算检查

### 6.2 缓解措施
1. **提供商适配器**: 为每个提供商创建适配器处理 API 差异
2. **成本缓冲**: 预算检查时预留 10% 缓冲防止超支
3. **原子操作**: 使用文件锁确保预算更新的原子性
4. **降级策略**: 预算不足时自动切换到更便宜的模型
