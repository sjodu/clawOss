# 阿里云服务器部署指南

## 服务器配置
- **规格**: 1核2GB（约 ¥80/月）
- **系统**: Alibaba Cloud Linux 3.2104 LTS 64位
- **部署方式**: Docker（推荐）

---

## 一、服务器准备

### 1. 登录服务器
```bash
ssh root@你的服务器IP
```

### 2. 安装 Docker（Alibaba Cloud Linux 3 专用）
```bash
# 更新系统
yum update -y

# 安装 Docker
yum install -y docker

# 启动 Docker
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
```

### 3. 安装 Docker Compose
```bash
# 下载 Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

## 二、部署 ClawOSS

### 方式 1：从 GitHub 克隆（推荐）

```bash
# 1. 克隆项目
cd /opt
git clone https://github.com/sjodu/clawOss.git
cd clawOss

# 2. 创建 .env 配置文件
cat > .env << 'EOF'
# ===== LLM 配置 =====
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
LLM_API_KEY=你的DeepSeek_API_Key
LLM_BASE_URL=https://api.deepseek.com
LLM_COST_INPUT=0.14
LLM_COST_OUTPUT=0.28

# ===== 预算控制 =====
BUDGET_MAX_USD=5.0

# ===== GitHub 配置 =====
GH_TOKEN=你的GitHub_Token
GH_USERNAME=sjodu
GH_EMAIL=你的邮箱

# ===== OpenClaw 版本 =====
OPENCLAW_VERSION=latest
EOF

# 3. 启动服务
docker-compose up -d

# 4. 查看日志
docker-compose logs -f
```

### 方式 2：手动上传文件

```bash
# 在本地执行（上传项目到服务器）
cd /Users/jiangjiang/Downloads/ClawOSS-6-release
scp -r . root@你的服务器IP:/opt/clawOss/

# 然后在服务器上执行
cd /opt/clawOss
docker-compose up -d
```

## 三、配置防火墙（开放端口）

### 阿里云安全组配置
1. 登录阿里云控制台
2. 进入 **ECS 实例** → **安全组** → **配置规则**
3. 添加入方向规则：
   - **端口**: 18789
   - **协议**: TCP
   - **授权对象**: 0.0.0.0/0（或你的 IP）
   - **描述**: ClawOSS Dashboard

### 服务器防火墙配置
```bash
# 如果使用 firewalld
firewall-cmd --permanent --add-port=18789/tcp
firewall-cmd --reload

# 如果使用 iptables
iptables -A INPUT -p tcp --dport 18789 -j ACCEPT
service iptables save
```

## 六、性能优化（1核2GB 专用）

### 1. 限制 Docker 资源
编辑 `docker-compose.yml`：
```yaml
services:
  clawoss:
    deploy:
      resources:
        limits:
          cpus: '0.9'      # 限制 CPU 使用
          memory: 1500M    # 限制内存使用（留 500MB 给系统）
        reservations:
          cpus: '0.5'
          memory: 800M
```

### 2. 启用 Swap（防止 OOM）
```bash
# 创建 2GB Swap
dd if=/dev/zero of=/swapfile bs=1M count=2048
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' >> /etc/fstab

# 验证
free -h
```

### 3. 调整心跳周期（减少资源消耗）
编辑 `.env`：
```bash
# 从 5 分钟改为 10 分钟
HEARTBEAT_INTERVAL_MINUTES=10
```

## 七、故障排查

### 问题 1：Docker 启动失败
```bash
# 查看详细错误
docker-compose logs

# 检查端口占用
netstat -tlnp | grep 18789

# 重新构建镜像
docker-compose build --no-cache
docker-compose up -d
```

### 问题 2：内存不足
```bash
# 查看内存使用
free -h
docker stats

# 清理 Docker 缓存
docker system prune -a

# 重启服务
docker-compose restart
```

### 问题 3：Gateway 无法访问
```bash
# 检查防火墙
firewall-cmd --list-ports

# 检查安全组（阿里云控制台）
# 检查 Gateway 状态
docker-compose exec clawoss openclaw gateway status

# 查看 Gateway 日志
docker-compose exec clawoss cat /root/.openclaw/logs/gateway.err.log
```

### 问题 4：GitHub 认证失败
```bash
# 重新认证
docker-compose exec clawoss gh auth login --with-token < <(echo $GH_TOKEN)

# 验证认证
docker-compose exec clawoss gh auth status
```

## 十、安全建议

### 1. 修改 SSH 端口
```bash
# 编辑 SSH 配置
vi /etc/ssh/sshd_config
# 修改 Port 22 为其他端口（如 2222）

# 重启 SSH
systemctl restart sshd
```

### 2. 禁用 root 登录
```bash
# 创建普通用户
useradd -m -s /bin/bash clawoss
passwd clawoss

# 添加 sudo 权限
usermod -aG wheel clawoss

# 禁用 root SSH 登录
vi /etc/ssh/sshd_config
# 设置 PermitRootLogin no
```

### 3. 配置 fail2ban（防暴力破解）
```bash
yum install -y fail2ban
systemctl start fail2ban
systemctl enable fail2ban
```

### 4. 定期备份
```bash
# 备份配置和数据
tar -czf /backup/clawoss-$(date +%Y%m%d).tar.gz \
  /opt/clawOss/.env \
  /opt/clawOss/workspace/memory/ \
  /opt/clawOss/workspace/logs/

# 设置定时备份（每天凌晨 2 点）
crontab -e
# 添加：0 2 * * * /path/to/backup-script.sh
```
